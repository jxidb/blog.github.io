tsconfig.json 配置项

tsconfig.json 是 ts 编译器的配置文件，ts 编译器可以根据这个文件的信息来对代码进行编译

  • include:用来指定哪些 ts 文件需要编译

  • exclude:用来指定哪些 ts 文件不需要被编译

  • extends：定义被继承的文件

  • files:表示需要编译的文件

  • compilerOptions:ts 编译器的配置，很重要

    • target 属性:用来指定 ts 被编译时的 ES 版本

    • module 属性：用来指定使用的模块化的规范

    • lib：用来指定在 ts 中用到的第三方库，一般不动，如果是在 node 环境中运行，可以改一下

    • outDir：用来指定 ts 文件编译后存放的路径

    • outFile:用于将全局作用域中的代码合并到一个 js 文件中，如果有模块化的话，模块化设置只能为 amd 或 system

    • allowJs：是否对 js 文件进行编译，默认 false

    • checkJs 是否检查 js 代码是否符合规范，默认 false

    • removeComments：编译时是否移除注释

    • noEmit:不生成编译文件

    • noEmitOnError:当代码有错误时，不编译 ts 文件

    • alwaysStrict：用来设置编译后的文件是否使用严格模式，默认 false

    • noImplicitAny:不允许隐式 any 类型,默认 false

    • noImplicitThis:不允许不明确类型的 this

    • strictNullChecks：严格的检查空值

    • strict 所有严格检查的总开关。如果设置为 true，就代表打开了所有严格检查

      constructor(name: string, age: number) {
      this.name = name
this.age = age
console.log(this)
      }
      //实例方法中，this指向的就是当前调用的实例对象
